# SmartShower

Розумна черга в душові кімнати гуртожитку №1, побудована на ESP32 +
Telegram-бот'і + двох TM1637-таймерах + OLED-табло.

Замість того щоб збиратися натовпом у коридорі біля душу, ти стаєш у
чергу через бот або через велику фізичну кнопку, чекаєш сповіщення «твоя
черга наступна», заходиш у душ, відмічаєш температуру води, що тобі
сподобалась. OLED у коридорі завжди показує хто наступний, а 7-сегментні
дисплеї біля кожної кабінки — поточний таймер душу.

---

## Залізо

- **MCU:** ESP32-WROOM (`board = upesy_wroom`)
- **2× TM1637** 7-сегмент із двокрапкою — таймери для кожного душу
- **1× SSD1306 128×32 OLED** — спільне табло (стан черги)
- **2× фіксаційні кнопки** — на дверях/вимикачах кожного душу (LOW = зайнято)
- **1× фізична кнопка** — стати в чергу анонімно
- **2× пари LED** (червоний / зелений) — стан кожного душу
- **8× кнопок температури** — по 4 на душ (1/2/3/4 = «крижана»/«трохи холодніша»/«трохи тепліша»/«окріп»; вирішує користувач, нам байдуже)
- **1× buzzer** — звукова реакція на дії

## Структура коду

```
include/
  bot.hpp           # API бота (повідомлення + хелпери)
  logic.hpp         # глобальні екземпляри (extern)
  Shower.hpp        # один душ (кнопка + таймер + LED + температура)
  SmartShower.hpp   # черга + OLED + buzzer + інтеграція з ботом
  secrets.example.hpp  # шаблон конфігу
  secrets.hpp       # реальний конфіг (gitignored)
  timeutil.hpp      # обгортка над ESP32 time API з DST для Києва

src/
  main.cpp          # setup() + loop() + Wi-Fi + WDT + бот-task
  logic.cpp         # визначення глобалів
  Shower.cpp        # реалізація Shower
  SmartShower.cpp   # реалізація SmartShower (найбільший файл)
  bot.cpp           # обробка Telegram-команд
```

### Архітектура виконання

Два ядра ESP32 розділені за обов'язками:

- **Ядро 1 (`loop()`):** опитування кнопок, дебаунс, відмалювання OLED,
  buzzer, таймери дисплеїв. Дуже швидкий цикл (`delay(1)` між ітераціями).
- **Ядро 0 (FreeRTOS task `botTask`):** Telegram long-polling, відправка
  повідомлень. Може блокуватися на HTTP — це не зачіпає реакцію кнопок.

Міжядерна синхронізація:

- `SemaphoreHandle_t queueMutex` навколо будь-якого доступу до черги
- `volatile bool pendingNotifyNext` — ядро 1 просить ядро 0 надіслати
  повідомлення «твоя черга наступна»
- `volatile bool pendingBeep` + `pendingBeepIsError` — ядро 0 просить
  ядро 1 пікнути buzzer'ом (бо `tone()` має йти з одного потоку)

### Стійкість

- **Watchdog** (`esp_task_wdt`, 15с) на `loop()` — будь-яке зависання → reboot
- **WiFi timeout** 30с у `setup()` → `ESP.restart()`, плюс автоматичне
  перепідключення (`setAutoReconnect(true)`)
- **NVS-черга:** `Preferences` зберігає чергу після кожної модифікації.
  Reboot/світло — черга не втрачається
- **DST** через `configTzTime("EET-2EEST,M3.5.0/3,M10.5.0/4", ...)` —
  автоматичне перемикання літо/зима для Києва
- **NTP failsafe:** поки час не синхронізувався, система вважає що зараз
  робочий час (інакше до синку черга би автоматично чистилась)
- **Авто-вибуття:** якщо першого сповістили («твоя черга наступна»), а
  він за 5 хвилин не зайшов у душ — автоматично видаляємо і сповіщаємо
  наступного

## Збірка та заливка

### Вимоги

- PlatformIO (CLI або VS Code Extension)
- ESP32 з USB-кабелем

### Конфігурація

1. Скопіюй `include/secrets.example.hpp` у `include/secrets.hpp`
2. Заповни:
   ```cpp
   #define WIFI_SSID "..."
   #define WIFI_PASS "..."
   #define BOT_TOKEN "<token_від_BotFather>"
   #define ADMIN_ID  "<твій_telegram_id>"  // для /clear_queue
   #define GROUP_ID  "-100..."             // ID чату де бот працює
   ```
3. Перевір що піни відповідають твоєму монтажу

### Команди PlatformIO

```bash
# Зібрати
pio run -e upesy_wroom

# Залити на ESP32
pio run -e upesy_wroom -t upload

# Залити і одразу відкрити serial monitor
pio run -e upesy_wroom -t upload -t monitor

# Тільки монітор
pio device monitor
```

Якщо `pio` не у `$PATH` (fish shell):

```fish
fish_add_path ~/.platformio/penv/bin
```

## Telegram-команди

Усі команди працюють лише в груповому чаті (`GROUP_ID`).

| Команда | Що робить |
|---------|-----------|
| `/start` | Привітання та список команд |
| `/get_info` | Стан обох душів (температура, користувач, час) + довжина черги |
| `/queue` | Повний список черги з іменами |
| `/position` | Твоя позиція в черзі |
| `/join_to_queue` | Додатися в чергу |
| `/leave_from_queue` | Вийти з черги |
| `/clear_queue` | (тільки `ADMIN_ID`) Очистити чергу повністю |

Команди підтримують суфікс `@BotName` (Telegram автоматично додає його у
групах із багатьма ботами).

## Фізичні елементи керування

- **Кнопка кожного душу** — фіксаційна. LOW = душ зайнятий, HIGH = вільний.
  Натискання при непустій черзі автоматично «приймає» першого в черзі і
  сповіщає наступного. При пустій черзі — просто «зайнято кнопкою»
  (анонім).
- **Queue button** — натискання додає анонімного користувача (`id="0"`)
  у чергу. Дебаунс 2с.
- **Кнопки температури** — фіксують вибране користувачем значення (1-4).
  Зберігаються разом з його ID та часом, видно через `/get_info`.

## Конфіг неробочого часу

У `include/secrets.hpp`:

```cpp
#define NIGHT_TIME_START 0        // ніч від 00:00
#define NIGHT_TIME_FINISH (6-1)   // до 05:59 включно (6-1=5)
#define MIDDAY_TIME_START 11      // обід від 11:00
#define MIDDAY_TIME_FINISH (14-1) // до 13:59 включно
```

У неробочий час OLED показує анімацію `OFF TIME`, бот відмовляє у
`/join_to_queue`, фізична кнопка черги ігнорується. Черга автоматично
чиститься.

## Серіал-логи

Усі важливі події логуються у `Serial` (115200 baud):

```
[Shower 1] init OK, busy=0
[Init] S1 T1 pin=15 -> HIGH (idle)
[Input] S2 T3 pressed
[Shower 1] T=2 user=@vova at 14:23:11
[Buzzer] OK
[Queue] auto-removing 1433760480 after 300s timeout
[Bot] sendMessage failed
```

Префікси: `[main]`, `[WiFi]`, `[Shower N]`, `[Init]`, `[Input]`,
`[Buzzer]`, `[Queue]`, `[Bot]`, `[NVS]`.

## Що ще можна зробити (нічого не блокує)

- Веб-морда (HTTP-сервер на ESP32 зі сторінкою статусу)
- OTA-оновлення прошивки через WiFi
- Сторінкова ротація OLED (черга / стан душу 1 / стан душу 2)
- Статистика (кількість душів за день, середній час)
- MQTT-інтеграція для Home Assistant
- Розблокування 3 кнопок температури (після фіксу заліза)
- Long-press queue button для виходу з черги без бота
